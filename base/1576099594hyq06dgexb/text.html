SPI


SPI - синхронный последовательный интерфейс когда данные передаются по одному биту в моменты времени, определяемые тактовым сигналом

В SPI используется 4 проводника:

->SCK для передачи тактовых сигналов (Source Clock)

->MISO для приёма данных

->MOSI для отправки данных

->SS для выбора ведомого устройства (Select Slave)


Master Slave

+===========+ +=======+

| MCU SCK |----->| SCK MCU |

| MISO|<-----|MISO |

| MOSI|----->|MOSI |

| __ | | __ |

| SS |------>| SS |

+===========+ +=======+


Интерфейс организован по принципу ведущий-ведомый (master-slave)

Такой способ обмена данными называется полнодуплексным, когда 2 устройства одновременно передают данные друг другу по 2-м разным проводникам


Линия MOSI используется для выдачи сигнала из ведущего ус-ва (Master Output) и одновременно вход для ведомого (Slave Input), а линия MISO - вход ведущего (Master Input) и выход ведомого (Slave Output)


Тактовые импульсы, которые задают момент обмена данными, формирует ведущее ус-во


Порядок обмена данными для SPI


Master Slave

генератор SPI +=======+ +=======+

| |MCU SCK |------>| SCK>--MCU |--------------------------|

||<----------------------+------<MISO|<------|MISO<--------+--<| |

-->[рег.данных]>+------>MOSI|------>|MOSI>--------+-->[рег.данных]<-

[рег.управления]| __ | | __ |

[рег.состояния] | SS |----->| SS |

+======+ +=======+

Заранее в ведущем ус-ве записывается информация в регистр управления о том как будет осуществляться обмен данными:

->Скорость передачи

->Момент тактового импульса в который происходит обмен

->Порядок выхода битов из регистра данных

etc


Обмен по SPI начинается с загрузки данных в рег. данных master


Генератор SPI master формирует последовательность из 8-и тактовых импульсов


При переходе сигнала из лог. 0 в лог. 1 {или наоборот} на SCK master выталкивает на MOSI младший{или старший} разряд своего регистра данных и одновременно ведомое уствройство выталкивает младший{или старший} разряд из своего регистра данных


При переходе из лог. 1 в лог. 0 {или наоборот} SCK ведущее ус-во записывает сигнал с MISO в старший{или младший} бит регистра данных, а ведомое с MOSI




Перед началом передачи master должен установить SS в 1, а после окончания в 0

Наиболее часто используется параллельное подключение при котором SCK MOSI MISO всех ведомых подключаетюся к SCK MOSI MISO ведущего

В этом случае ведущее ус-во должно иметь дополнительные линии для управления SS всех ведомых ус-в
